🧠 Sabse Pehle — Redux Ka Basic Idea

Redux me 3 main cheezein hoti hain:

Store → data ka ghar

Action → kya change karna hai

Reducer → kaise change karna hai

🔥 Reducer Kya Hota Hai?
Simple Definition:

Reducer ek function hota hai jo old state aur action leta hai, aur new state return karta hai.

Syntax:
(state, action) => newState
🏠 Real Life Analogy

Soch:

Store = Bank account 💰

Action = "500 rupees deposit"

Reducer = Bank ka rule system jo decide karega balance kaise update hoga

Reducer khud se kuch nahi karta.
Jab action aata hai tab decide karta hai state kaise change hogi.

Example Without Toolkit (Old Redux)
const initialState = { count: 0 }

function counterReducer(state = initialState, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 }
    default:
      return state
  }
}

Yaha:

state = current value

action.type = kya karna hai

return new state

⚠️ Important:
Reducer directly state change nahi karta
Wo naya object return karta hai

🚀 Ab Redux Toolkit Samjho Proper Flow Me

Redux Toolkit basically Redux ko simple karta hai.

1️⃣ createSlice() (Sabse Important)

Ye:

reducer banata hai

action creators banata hai

action types banata hai

Sab ek jagah.

Example
import { createSlice } from "@reduxjs/toolkit"

const counterSlice = createSlice({
  name: "counter",
  initialState: { value: 0 },
  reducers: {
    increment(state) {
      state.value += 1
    },
    decrement(state) {
      state.value -= 1
    }
  }
})
Yaha Kya Hua?

increment ek reducer function hai

Lekin tu direct state change kar raha hai 😳

Ye allowed hai kyunki RTK Immer use karta hai

Behind the scenes:
Wo copy banata hai aur immutable update karta hai.

Automatically Milta Hai:
counterSlice.reducer
counterSlice.actions.increment

Matlab:

dispatch(counterSlice.actions.increment())
2️⃣ configureStore()

Ye store banata hai.

import { configureStore } from "@reduxjs/toolkit"

const store = configureStore({
  reducer: {
    counter: counterSlice.reducer
  }
})

Yaha multiple slice bhi add kar sakte ho.

🧠 Sabse Simple Definition

dispatch ka matlab hai — action ko store tak bhejna

Matlab:

“Ye kaam karna hai” bolke Redux ko inform karna.

🏠 Real Life Analogy

Soch:

Store = Office 🏢

Reducer = Office ka rule system

Action = Application form

dispatch = Application submit karna 📩

Jab tak tu dispatch nahi karega,
tab tak reducer ko pata hi nahi chalega ki kuch change karna hai.

🔥 Technical Definition

dispatch() ek function hai jo:

Action ko store tak bhejta hai

Store us action ko reducer ko deta hai

Reducer new state return karta hai

UI update ho jati hai

📦 Example
Action
{ type: "increment" }
Dispatch
dispatch({ type: "increment" })

Ya Redux Toolkit me:

dispatch(increment())

🎯 Important Point

⚠️ Dispatch ke bina state kabhi change nahi hoti.

Redux me:

Direct state change allowed nahi.

Sirf dispatch ke through hi change hoti hai.

💡 Ek Strong Analogy

Soch state ek locker hai 🔐

Tu directly locker nahi khol sakta.
Tu guard ko bolta hai (dispatch).

Guard rules check karta hai (reducer).
Phir locker update hota hai (state change).

🧠 Redux Toolkit Example
const { increment } = counterSlice.actions

Yaha kya ho raha hai?

increment() → action create karta hai

dispatch() → us action ko bhej raha hai

reducer → state update kar raha hai

lekin state kabhi change nahi hogi.

Dispatch = trigger 🔥

🧠 Exact Flow Kya Hota Hai?

Jab tu karta hai:

dispatch(addTodo({ text: "Study" }))

Tab internally ye hota hai:

Store ke paas current state hoti hai
        ↓
dispatch(action)
        ↓
Reducer(currentState, action)
        ↓
Reducer return karta hai newState
        ↓
Store old state ko replace kar deta hai
🔥 Important Line

Reducer ko hamesha current state milti hai,
aur reducer hi new state create karta hai.

Dispatch sirf action bhejta hai.

🏠 Analogy Time

Soch:

Store = diary 📘

Current state = diary ka last page

Dispatch = “naya entry likhni hai” bolna

Reducer = entry likhne ka rule

Reducer diary ka last page dekhta hai (current state),
phir new entry add karta hai,
aur updated diary wapas de deta hai.

🧩 Code Visualization

Reducer ka real structure hota hai:

function reducer(state, action) {
  // state = current state
  // action = kya karna hai
  return newState
}

Dispatch kabhi state ko touch nahi karta.

Dispatch ka kaam sirf:

store.dispatch(action)
🎯 Ek Line Me Yaad Kar

Dispatch → Action bhejta hai

Reducer → Current state + action se new state banata hai

Store → New state save karta hai
