🔹 Step-by-Step Flow
1️⃣ Provider ke andar state bana
const [user, setUser] = useState(null);

Yaha:

user = data

setUser = us data ko update karne ka function

2️⃣ Provider ne dono share kar diye
<UserContext.Provider value={{ user, setUser }}>

Matlab:

“Jo bhi mere andar hai, tum sab user aur setUser use kar sakte ho.”

3️⃣ Dusre component me use kiya
const { setUser } = useContext(UserContext);

Ab tumhe wahi setUser function mil gaya
jo originally Provider ke andar bana tha.

4️⃣ Call kiya
setUser({ name: "Yash" });

Ab kya hua?

Provider ke andar wala user state update hua

React re-render karega

Jahan jahan user use ho raha hai wahan update dikhega

🔥 Important Concept

setUser child ka function nahi hai ❌
setUser parent (Provider) ka function hai ✔

Child bas usko use kar raha hai
kyunki Provider ne access de diya.

🧠 Strong Analogy

Provider = Main water tank 💧
setUser = Valve control

Ghar ke rooms (children)
valve ghuma sakte hain
kyunki tank ne control allow kiya hai.

Tum puch rahe ho:

<UserContextProvider>

   <Login/>
   <Profile/>

Isme ye sab Provider ke andar kyun dala?

Chalo seedha jawab 👇

🔹 Simple Reason

Kyuki:

Login ko setUser chahiye

Profile ko user chahiye

Aur ye dono UserContext.Provider ke value me diye gaye hain.

Isliye inhe Provider ke andar rakhna zaroori hai.

🔥 Agar bahar rakhte to?
<h1>React with Chai</h1>

<Login/>

<UserContextProvider>
   <Profile/>
</UserContextProvider>

Ab:

Profile ko context milega ✔

Login ko nahi milega ❌

Kyuki Login Provider ke bahar hai.

🔹 Toh Provider ka kaam kya hai yaha?

Ye define karta hai:

“Context ka scope yahan se start hota hai.”

Jo andar hai → access karega
Jo bahar hai → access nahi karega

🔥 H1 ko kyun dala?

Honestly 😄
<h1> ko context ki zarurat nahi hai.

Wo sirf example me dala hai.

Tum chaho to aise bhi kar sakte ho:

<h1>React with Chai</h1>

<UserContextProvider>
   <Login/>
   <Profile/>
</UserContextProvider>

Bilkul valid hai.

🧠 Strong Analogy

Provider = Ek WiFi router 📡

Jo devices router ke range me hain
wahi internet use kar sakte hain.

Router ko jitna bada area cover karwana hai
utna hi area wrap kar do.

🔥 Real World Rule

Usually Provider ko top level pe wrap karte hain:

<UserContextProvider>
   <App/>
</UserContextProvider>

Taaki pura app access kar sake.

💎 Final Lock

Provider ke andar sirf wahi components rakho jinko us context ki zarurat hai.
Baaki components ko wrap karna mandatory nahi hai.
